# Work CheckList

항상 화면 위에 떠 있는 **가벼운 업무 체크리스트 위젯**. Windows Sticky Notes로 일과를 관리할 때 겪는 불편(다른 창을 띄우면 메모가 뒤로 내려감, 공간 부족)을 보완하려고 만든 도구입니다.

- 항상 위에 고정 + 크기 조절 가능, 마지막 크기·위치 기억
- 업무는 **빈 체크리스트로 시작** → 항목·하위 항목을 직접 추가하거나 **프리셋**으로 채움
- 항목 추가·삭제·**드래그 정렬**, 업무 단위 우선순위 정렬
- 유형(초기개발/유지보수)·상태(진행/완료/중단/종료/제외) 태그
- **[⚙ 설정]** — 일반 설정(항상 위·글꼴·글자 크기·테마·데이터 폴더 열기)과 **프리셋 관리**(공통/카테고리별 항목·카테고리 직접 편집)를 한 폼에서
- **완전 로컬 · 외부 의존성 0** (파이썬 표준 라이브러리 `tkinter` + `json`만 사용)

---

## 요구 사항

- **Python 3.x** (`tkinter` 포함 — 대부분의 공식 배포판에 기본 포함)
- 외부 패키지 설치 불필요

> `tkinter`가 없는 환경에서는 안내 메시지를 출력하고 종료합니다.

## 실행

```bash
python checklist.py
```

데이터 파일(`checklist_data.json`)은 `checklist.py`와 같은 폴더에 자동 생성·관리됩니다.

콘솔 창 없이 더블클릭으로 실행하려면 `pythonw checklist.py` 바로가기를 만들거나, `pyinstaller --onefile --windowed checklist.py`로 단일 실행 파일을 만들 수 있습니다.

## 기능

### 창
- **항상 위(always on top)** 토글 (기본 ON, [⚙ 설정 → 일반 설정]) — 다른 창을 띄워도 뒤로 내려가지 않음
- **크기 조절** 가능 (너비 400–560, 높이 자유) — 단, **데스크톱 작업 영역(작업 표시줄 제외)을 넘지 못하게 자동 제한**. 더 크게 늘리려 하면 작업 영역 크기에 맞춰지고, 다른(더 큰) 모니터에서 저장된 크기·위치도 현재 화면 안으로 보정해 복원
- 최대화 버튼/제목줄 더블클릭 최대화는 비활성화 (위젯이 화면을 덮지 않도록)

### 설정 ([⚙ 설정])
- **일반 설정** — 항상 위 토글 · **글꼴 / 글자 크기 / 테마** 변경(즉시 반영) · **데이터 폴더 열기** · **프리셋 기본값으로 초기화**
- **프리셋 관리** — 좌측 그룹 목록(`공통` + 카테고리)에서 고른 뒤 우측에서 항목을 **추가·수정·삭제·정렬**. 카테고리는 **추가·이름변경·삭제** 가능(이름 변경 시 기존 업무의 카테고리·항목 칸 참조도 함께 갱신)

### 전체 진행률
- 상단에 모든 업무를 합산한 진행률 바 (`전체 진행률 12/30 (40%)`)
- 상태가 `종료`/`제외`인 업무는 합산에서 제외

### 업무(블록)
- **제목 · 유형(초기개발/유지보수) · 카테고리 · 개수**를 정해 생성. 개수 N이면 같은 설정으로 N개 생성
- 헤더에 **유형 배지**와 **상태 콤보박스**(진행/완료/중단/종료/제외)
- **드래그 핸들(↕)** 로 업무 순서(우선순위) 변경 — 표시 번호 `#N`은 위치 기준으로 자동 재계산
- 접기/펼치기(▼/▶), 수정(✎), 삭제(🗑)

### 항목 / 하위 항목
- 업무는 **빈 상태로 시작**. `[+ 항목]`으로 항목 추가, 각 항목의 `[+세부]`로 하위 항목(2단계) 추가
- 항목·하위 항목 체크/해제, 삭제(✕)
- 항목 **드래그(↕) 정렬** (업무 내부)
- `[프리셋 불러오기 ▾]` — `공통` 및 카테고리별 `초기개발`·`유지보수` 기본 목록을 골라 채움
- 한 업무에 **여러 카테고리 프리셋**을 불러오면 `[DB작업]` / `[API작업]` 칸으로 자동 구분 (단일/자유 항목이면 헤더 없이 평탄하게)

### 저장
- 모든 변경은 즉시 로컬 JSON에 자동 저장
- 손상 시 백업 후 빈 상태로 복구, 저장 실패 시 메모리 데이터 보존 안내
- 읽기/쓰기 UTF-8 고정, 원자적 저장(`os.replace`)

## 파일 구조

| 파일 | 설명 |
| --- | --- |
| `core.py` | **공통 로직** — 상수 · 데이터 모델 · 영속성(load/save) · 마이그레이션 · 진행률 집계. **GUI 라이브러리를 일절 import하지 않음** (화면을 다른 스택으로 다시 만들어도 그대로 재사용 가능) |
| `checklist.py` | **UI 레이어** — tkinter 위젯 · 렌더링 · 이벤트 · 드래그 정렬. `core`에서 import해 사용 |
| `checklist_data.json` | 업무·항목 저장 파일 (실행 시 자동 생성) |

> 코드는 **"UI와 무관한 공통 로직(`core.py`)"** 과 **"화면(`checklist.py`)"** 으로 분리되어 있습니다. 데이터·저장·마이그레이션은 `core.py`에 모여 있어, 향후 화면을 다른 방식으로 다시 만들더라도 핵심 로직은 재사용할 수 있습니다.

프리셋(공통/카테고리별 항목)은 **데이터 파일(`checklist_data.json`)의 `presets` 블록**에 저장되며, **[⚙ 설정 → 프리셋 관리]**에서 편집합니다. `core.py`의 `DEFAULT_COMMON_ITEMS` · `DEFAULT_CATEGORY_TEMPLATES` 상수는 **최초 실행 시 시드값**이자 [프리셋 초기화] 시 되돌릴 기본값입니다.

## 데이터 모델 (`checklist_data.json`)

```json
{
  "version": 3,
  "next_id": 9,
  "always_on_top": true,
  "geometry": "360x460+100+80",
  "settings": {
    "font_family": "Segoe UI",
    "font_size": 10,
    "theme": "vista"
  },
  "presets": {
    "common": ["요구사항 확인", "빌드 확인"],
    "category_order": ["화면개발", "DB작업"],
    "categories": {
      "화면개발": { "초기개발": ["화면 UI/UX 구현"], "유지보수": ["기존 동작 회귀 테스트"] }
    }
  },
  "tasks": [
    {
      "id": 8,
      "title": "회원 API",
      "kind": "유지보수",
      "categories": ["DB작업", "API작업"],
      "status": "진행",
      "collapsed": false,
      "next_item_id": 5,
      "items": [
        {
          "id": 1, "text": "응답 포맷 확인", "checked": false,
          "group": "API작업",
          "subitems": [{ "id": 2, "text": "에러 코드 표준화", "checked": false }]
        }
      ]
    }
  ]
}
```

| 필드 | 의미 |
| --- | --- |
| `geometry` | 마지막 창 크기·위치 (`"360x460+100+80"`) |
| `settings` | 일반 설정: `font_family` / `font_size` / `theme` |
| `presets` | 편집 가능한 프리셋: `common`(공통 항목) / `category_order`(카테고리 표시 순서) / `categories`(카테고리별 `초기개발`·`유지보수` 항목) |
| `tasks[]` 순서 | 화면 표시 순서 = 우선순위 (드래그로 변경) |
| `task.kind` | 유형: `초기개발` / `유지보수` / `""` |
| `task.status` | 상태: 진행 / 완료 / 중단 / 종료 / 제외 |
| `item.group` | 카테고리 칸 구분용 (`""`이면 자유 항목) |
| `item.subitems` | 하위 항목(2단계) |

> v1(옛 구조) 데이터는 실행 시 자동으로 v2로 변환됩니다. 변환 시 **체크돼 있던 항목과 직접 작성한 항목만 보존**하고, 체크 안 된 자동 템플릿 항목은 비웁니다(기본 빈 정책).
>
> v2 → v3: 실행 시 `settings`·`presets` 블록이 코드 기본값으로 자동 시드됩니다(기존 업무·항목은 그대로 유지).

## 설계 배경

이 도구는 "입력 의미를 분석해 항목을 생성"하는 똑똑한 도구가 아니라, **일관성과 누락 방지**에 가치를 둡니다. 초기에는 카테고리 선택 시 항목을 자동 생성하는 규칙 기반 방식이었으나, 실제로 쓰다 보니 자동 항목이 부담스럽고 업무마다 필요한 항목이 달라 — **기본은 비우고, 필요할 때 프리셋으로 채우고, 자유롭게 편집**하는 현재 구조로 정착했습니다.

### 하지 않는 것 (비범위)
- 입력 문장의 의미 분석 / 자동 분해
- 외부 서버 전송 / 클라우드 동기화 (전부 로컬)
- 다중 사용자 / 협업
- 되돌리기(undo) (삭제는 확인 후 진행)
